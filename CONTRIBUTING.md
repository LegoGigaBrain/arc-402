# Contributing to ARC-402

ARC-402 is an open standard in active development. Contributions are welcome.

## What We're Looking For

- **Spec feedback** — gaps, ambiguities, edge cases in the five primitives
- **Implementation feedback** — practical blockers when implementing against the spec
- **Reference implementation** — improvements to the Solidity contracts or TypeScript SDK
- **Real-world implementations** — if you're building on ARC-402, open an issue to get listed

## How to Contribute

1. Open an issue first for significant changes
2. Fork, branch, PR
3. For spec changes: include motivation and any relevant prior art
4. For implementation changes: include tests

## Spec Change Process

Spec changes are classified as:
- **Editorial** (typos, clarity) — direct PR
- **Non-breaking addition** (new SHOULD/MAY, new examples) — issue + PR
- **Breaking change** (modifying MUST, changing schemas) — issue, discussion period, then PR

## Status

`v1` — The protocol is live on Base mainnet. Breaking changes follow the change process above.
